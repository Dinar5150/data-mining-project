\section{Data Understanding}

\subsection{Collect Initial Data}

\subsubsection*{Initial Data Collection}
The raw dataset is \texttt{enriched\_prs\_raw.jsonl}. Candidate pull requests were first selected from GH Archive by finding merged pull request events and joining them with issue comment, review, and review comment activity for the same repository and pull request number.

The candidate filter kept pull requests with 2--30 changed files, 50--2000 changed lines, at least two issue comment events, at least one review event, and at least two review comment events. Each candidate was then enriched through the GitHub REST API with pull request metadata, changed files, formal reviews, inline review comments, pull request comments, the full unified diff, and linked issue data when available. Linked issues were detected from closing-keyword references such as \texttt{fixes \#123} in the pull request title or body.

\subsection{Describe Data}

\subsubsection*{Data Description}
The collected raw file contains 7,563 enriched pull request records and occupies about 1.2 GB. It covers 4,758 repositories and 6,529 authors. Most pull requests were created and merged in March 2025; enrichment was retrieved on April 30, 2026 and May 1, 2026.

Each raw record contains the candidate row, pull request object, changed-file list, formal reviews, inline review comments, pull request comments, full diff, linked issue, linked issue comments, API errors, and retrieval timestamp. The current raw layer is enriched data, not yet the final accepted dataset.

\subsection{Verify Data Quality}

\subsubsection*{Quality Verification}
Raw enrichment quality is sufficient for analysis, but the file is not yet the final curated dataset. Pull request objects, file lists, and full diffs are present in 100\% of records. Formal reviews are present in 99.9\%, and review comments are present in 99.7\%.

The main quality limitation is linked issue coverage: only 924 records, or 12.2\%, contain an explicit linked issue resolved by the current regex. Under the current strict filters, 721 records pass as accepted traces, equal to 9.5\% of the raw enriched file. The dominant rejection reason is \texttt{no\_explicit\_linked\_issue}, followed by missing source-code patches or source files. The next preparation stage should therefore either preserve the strict linked-issue requirement and expand collection volume, or revise the linked-issue extraction rule if higher recall is required.
